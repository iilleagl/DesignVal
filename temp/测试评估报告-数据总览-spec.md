# 测试评估报告 — 数据总览页面 — 设计需求文档

本文档描述「测试评估报告 - 数据总览」页面的详细设计需求，用于指导 Pencil 设计工具基于组件库 `designsystem/devUI/components/devUI2.pen` 进行设计实现。风格参考华为云设计规范与 CodeArts 产品，浅色系、白底、干净、多细节。

---

## 1. 整体布局与视觉风格

**布局结构**：顶栏（全局导航）+ 左侧栏（工具链/产品导航）+ 主内容区。主内容区为白底页面导航区（面包屑、标题区、页签）+ 数据展示区（图表与卡片网格）。

**视觉风格**：
- 色调：浅色系；主内容区与卡片为白底（#ffffff），图表区可用浅绿渐变（如 #e8f5e9 → #f1f8e9）、浅灰分割（#f5f5f5）；强调色沿用 DevUI 品牌色与状态色（绿/蓝/橙/红）。
- 组件特征：圆角与阴影与 DevUI 一致；卡片带轻微阴影、分割线用 #dfe1e6；图表容器白底或浅色底，留白充足。
- 文字：主标题 16–20px、卡片标题 14px、正文/数据 12–14px；主文字 #252b3a、次要 #575d6c、辅助 #8a8e99；字体 Noto Sans SC / HarmonyOS Sans SC。
- 华为云 CodeArts 风格：信息层级清晰、留白适中、图标与数据并重、表格与图表区对齐规整。

**画布尺寸**：1920×1080（或 1440×900 主内容可视区）。

## 2. 核心区域详解

### A. 顶部全局导航栏

- 位置：页面最顶部，全宽，固定高度约 56px。
- 内容：Logo/产品名 + 主导航菜单 + 右侧（通知、帮助、用户头像）。
- 交互：当前产品/模块高亮；菜单项 hover/active 态。

**内容物料**：

| 元素 | 类型 | 具体内容 |
|------|------|----------|
| Logo/产品名 | 文本 | CodeArts |
| 菜单项1 | 文本 | 首页 |
| 菜单项2 | 文本 | 项目 |
| 菜单项3 | 文本 | 测试 |
| 菜单项4 | 文本 | 流水线 |
| 右侧通知 | 图标 | 铃铛 |
| 右侧帮助 | 图标 | 问号 |
| 用户区 | 文本 | 当前用户 |

**还原要点**：使用 devUI 顶部导航栏组件（如 Rpv7B），ref 引用后通过 descendants 替换为上述文案与图标。

---

### B. 左侧导航栏

- 位置：顶栏下方、主内容区左侧，宽度约 240px（展开态）。
- 内容：项目/空间选择器 + 一级菜单（图标+文字）+ 二级菜单；当前页对应菜单项高亮。
- 交互：展开/收起；一级可展开折叠；选中态背景与文字强调。

**内容物料**：

| 元素 | 类型 | 具体内容 |
|------|------|----------|
| 项目选择器 | 文本 | Phoenix-Sample 或 当前项目名 |
| 一级菜单1 | 文本+图标 | 代码托管 / 代码管理图标 |
| 　├ 二级 1-1 | 文本 | 仓库列表 |
| 　├ 二级 1-2 | 文本 | 分支管理 |
| 一级菜单2 | 文本+图标 | 流水线 / 流水线图标 |
| 一级菜单3 | 文本+图标 | 测试 / 测试图标 |
| 　├ 二级 3-1 | 文本 | 测试计划 |
| 　├ 二级 3-2 | 文本 | 测试报告（当前选中） |
| 　└ 二级 3-3 | 文本 | 数据总览（当前页，高亮） |

**还原要点**：使用 devUI 工具链侧栏（JGEqF 展开态），按二级菜单数量选对应变体；ref + descendants 填菜单文案。

---

### C. 页面导航区（白底）

整块为白色背景，包含三行：面包屑、标题行、页签行。与主内容区左右 padding 一致（如 24px）。

#### C1. 第一行 — 面包屑

- 位置：主内容区顶部第一行。
- 内容：层级导航，3 层；最后一级为当前页不链接。

**内容物料**：

| 元素 | 类型 | 具体内容 |
|------|------|----------|
| 层级1 | 文本 | 首页（可点击） |
| 层级2 | 文本 | 测试（可点击） |
| 层级3 | 文本 | 测试评估报告 - 数据总览（当前页） |

**还原要点**：面包屑组 devui:iLVzA（3 层），descendants 填上述三层文案。

---

#### C2. 第二行 — 返回 + 图标 + 标题 + 基本详情信息

- 位置：面包屑下方；与标题同一行左对齐，右侧可放基本详情（如报告名称、时间范围、环境等）。
- 布局：左侧「返回」按钮 + 图标 + 主标题；右侧为基本详情信息（多行或单行均可，小字号、次要色）。

**内容物料**：

| 元素 | 类型 | 具体内容 |
|------|------|----------|
| 返回按钮 | 文本 | 返回 |
| 图标 | 图标 | 报告/文档类图标 |
| 主标题 | 文本 | 测试评估报告 - 数据总览 |
| 详情项1 | 文本 | 报告名称：Phoenix 迭代1 评估报告 |
| 详情项2 | 文本 | 时间范围：2025-03-01 ~ 2025-03-15 |
| 详情项3 | 文本 | 环境：预发环境 |

**还原要点**：头信息区可参考 Xxf4j 或自建一行：返回按钮（qTc8e）+ 图标 + 标题（16–20px）；右侧用 12/14px 次要色文案排列。

---

#### C3. 第三行 — 页签（3 个按钮，靠右对齐，与标题横向对齐）

- 位置：标题行下方；页签与标题区域同一水平带，页签整体靠右。
- 内容：3 个页签项；第一个为「数据总览」选中态，其余为「用例分析」「缺陷分析」等。

**内容物料**：

| 元素 | 类型 | 具体内容 |
|------|------|----------|
| 页签1 | 文本 | 数据总览（选中） |
| 页签2 | 文本 | 用例分析 |
| 页签3 | 文本 | 缺陷分析 |

**还原要点**：使用 devUI 页签 Tabs 3 项（hbi7t）或 Pills 3 项（cee0f）；整块容器 flex 靠右，与标题行对齐。

---

### D. 数据区 — 第一行（两列）

#### D1. 第一块：雷达图 + 数据结论

- 布局：左侧一张 5 边形雷达图，浅绿色渐变底色（如 #e8f5e9 到 #f1f8e9）；右侧白底卡片为「雷达图数据结论」文字说明。
- 尺寸：左侧图表区约 50% 宽或固定比例；右侧结论区 50%，内边距 16–20px。

**内容物料**：

| 元素 | 类型 | 具体内容 |
|------|------|----------|
| 雷达图标题 | 文本 | 质量维度概览（或 评估维度分布） |
| 雷达图 5 维度 | 文本/图例 | 功能覆盖、性能、可靠性、安全、可维护性（或业务给定维度名） |
| 结论标题 | 文本 | 数据结论 |
| 结论正文1 | 文本 | 功能覆盖与可靠性得分较高，整体达标。 |
| 结论正文2 | 文本 | 性能与安全维度有提升空间，建议重点优化。 |

**还原要点**：左侧用 frame + 浅绿渐变 fill + 内嵌 5 边形雷达示意（或占位图表）；右侧为白底卡片（7ipso 或自建），内多行正文。

## 3. 组件使用与还原要点

实现时优先从 `designsystem/devUI/components/devUI2.pen` 选用组件，复制到设计稿后通过 ref 引用；引用后必须修改实例内的文案、图标、占位内容以符合本页业务语义。

| 区域 | 建议组件 | 说明 |
|------|----------|------|
| 顶栏 | 顶部导航栏 Rpv7B | 替换菜单项与右侧区域文案/图标 |
| 左侧栏 | 工具链侧栏 JGEqF | 按二级菜单数量选变体，填项目名与菜单文案 |
| 面包屑 | 面包屑组 iLVzA | 3 层文案 |
| 标题行 | 头信息 Xxf4j 或自建 | 返回按钮 qTc8e + 图标 + 标题 + 右侧详情 |
| 页签行 | 页签 Tabs hbi7t 或 Pills cee0f | 3 项，整体靠右 |
| 雷达图块 | 基本卡片 7ipso + 自建图表区 | 浅绿渐变底 + 右侧白底结论卡片 |
| 6 数据卡 | 基本卡片 7ipso ×6 | 图标+标题+数据，网格布局 |
| 折线图卡片 | 基本卡片 7ipso | 标题「响应码分布」+ 折线占位 + 图例 |
| 圆盘图卡片×2 | 基本卡片 7ipso | 用例通过情况 / 缺陷严重程度 + 圆环占位 + 图例 |

**设计规范**：颜色使用 Design Token（#252b3a、#575d6c、#8a8e99、#ffffff、#f5f5f5、#e8f5e9 等）；字号 12/14/16/20/24 px；间距 4 的倍数；圆角与阴影与组件库一致。

---

## 4. 实现顺序建议

1. 搭整体框架：顶栏 + 侧栏 + 主内容区（白底容器），保证层级与占位正确。
2. 主内容区上部：面包屑 → 标题行（返回+图标+标题+详情）→ 页签行（3 项靠右）。
3. 数据区第一行：左侧雷达图块（浅绿渐变+5 边形）+ 右侧结论卡片；右侧 6 张数据卡片网格。
4. 数据区第二行：折线图卡片（响应码分布）+ 用例通过情况圆盘图 + 缺陷严重程度圆盘图。
5. 细节：所有 ref 组件的 descendants 填业务文案；图表用占位 frame/形状+图例；统一间距与对齐；截图验证后按常见错误清单审核修复。
