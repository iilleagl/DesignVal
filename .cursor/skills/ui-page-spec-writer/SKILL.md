# UI 页面设计文档生成

根据用户的设计需求，输出结构化的页面设计规范文档。文档将整页拆解为可执行的局部设计任务，供 `pen-design-generator` 消费。

## 工作流程

### 1. 需求收集

从用户输入中提取以下信息（缺失项主动追问）：

| 信息 | 必须 | 说明 |
|------|------|------|
| 页面用途 | 是 | 如：控制台仪表盘、表单配置页、数据列表页 |
| 核心功能/内容 | 是 | 页面需要展示或操作哪些信息 |
| 目标用户/平台 | 否 | Web/移动端；管理后台/面向用户 |
| 参考页面/截图 | 否 | 若有则优先参考 |
| 特殊要求 | 否 | 品牌色、固定布局约束等 |

### 2. 页面结构规划

根据需求确定页面的宏观布局模型。常见模型：

```
经典三栏：顶栏 + 左侧栏 + 主内容区
仪表盘型：顶栏 + 左侧栏 + 主内容区 + 右侧栏
表单型：  顶栏 + 面包屑 + 居中表单卡片
列表型：  顶栏 + 头信息 + 筛选/搜索 + 表格 + 分页
```

### 3. 区块拆解

将页面拆解为**有序的区块列表**，每个区块作为一个独立的设计任务。拆解原则：
- 按视觉/功能边界划分（如顶栏、侧栏、头信息区、内容区各独立）
- 复杂区域继续细分（如"主内容区"下分"筛选栏""数据表格""分页器"）
- 每个区块可独立实现，也可说明与其他区块的位置关系

### 4. 生成设计文档

按照下文的**文档模板**输出完整的 Markdown 设计文档。

```markdown

基于组件库 `designsystem/devUI/components/devUI2.pen` 进行设计实现。

---

**布局结构**：[描述宏观布局，如"顶栏 + 左侧栏 + 主内容区"，
说明各区域的位置关系和比例]

**视觉风格**：
- 色调：[主色调描述]
- 组件特征：[圆角、阴影、边框等]
- 文字：[主/次文字颜色、字体]

**画布尺寸**：[宽×高，如 1920×1080]

---

## 2. 核心区域详解

**内容物料**：（必须逐项列出该区块中所有需要填充的文本、图标、图片。此表是设计稿落地时**对组件内文案进行填充的唯一依据**——pen-design-generator 会按此表在 ref 的 `descendants` 中替换占位文案；顶栏、侧栏、头信息、表格等多文案组件尤其依赖本表填齐，不得用占位符。）

| 元素 | 类型 | 具体内容 |

| [如：菜单项1图标] | 图标 | [如：首页图标/home] |
| ... | ... | ... |

**还原要点**：[简要的实现提示]

---

### B. [区块名称]

...（每个区块一个小节，按从上到下、从左到右排序。
每个区块都**必须**包含「内容物料」表格。）

## 3. 组件使用与还原要点

实现时优先从 `designsystem/devUI/components/devUI2.pen` 选用组件，
复制到设计稿后通过 ref 引用；引用后**必须**修改实例内的文案、
图标、占位内容以符合本页业务语义。

| 区域 | 建议组件 | 说明 |
|------|----------|------|
| [区域名] | [组件名称] | [选型理由和定制要点] |
| ... | ... | ... |

**设计规范**：颜色使用 Design Token；字号 12/14/16/20/24/36 px；
间距为 4 的倍数；圆角与阴影与组件库一致。

---

## 4. 实现顺序建议

1. [先搭整体框架：顶栏+侧栏+主内容区分栏]
2. [再逐区块填充内容，按依赖顺序排列]
3. [最后处理细节：状态、图标、间距微调]
```

## 区块描述规范

每个区块的描述需涵盖以下维度（按实际需要取舍）：

| 维度 | 说明 | 示例 |
|------|------|------|
| **位置** | 在页面中的位置 | "页面顶部，全宽" |
| **尺寸** | 宽高约束 | "全宽×56px 固定高度" |
| **布局** | 内部排列方式 | "水平排列，左右分组" |
| **内容** | 具体包含哪些元素 | "Logo + 导航菜单 + 用户头像" |
| **内容物料** | 每个元素的具体文本/图标/图片 | 见下方物料表要求 |
| **交互** | 状态变化与响应 | "当前页菜单项高亮" |
| **组件** | 推荐使用的 DevUI 组件 | "顶部导航栏 `Rpv7B`" |

### 内容物料表要求（关键）

每个区块**必须**包含一个「内容物料」表格，逐项列出该区块中所有需要填充的文本、图标和图片。这是设计实现时替换组件默认占位内容的唯一依据。

**物料表格式**：

```markdown
| 元素 | 类型 | 具体内容 |
|------|------|----------|
| 元素名称 | 文本/图标/图片 | 实际值 |
```

## 质量检查清单

文档完成后自检：

- [ ] 整体布局有清晰描述，包含视觉风格
- [ ] 每个可见区块都有独立小节
- [ ] 每个区块都有「内容物料」表格，逐项列出文本/图标/图片的具体值
- [ ] 物料表中无占位符（如"菜单项1""文本内容"），全部为实际业务文案
- [ ] 组件推荐表覆盖了所有区块
- [ ] 实现顺序建议合理（先框架后内容）
- [ ] 文档可直接交给 `pen-design-generator` 使用
