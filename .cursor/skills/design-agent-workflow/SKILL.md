# 设计 Agent 完整流程

端到端的三阶段设计流程。**必须严格遵守** `.cursor/rules/design-system-usage.mdc`（组件优先、规范遵从）。

---

## 阶段 1：页面规范文档（Spec）

**使用技能**：`.cursor/skills/ui-page-spec-writer/`

1. 根据用户需求，调用 `ui-page-spec-writer` 生成完整的页面设计规范文档。
2. 文档必须包含：
   - 整体布局与视觉风格
   - 每个区块的详细描述
   - **内容物料表**：逐项列出所有文本、图标、图片的**具体业务文案**（禁止占位符）
   - **组件选型表**：每个区域映射到 DevUI 组件库的具体组件 ID
3. 输出的 Spec 文档是下一阶段的直接输入。

---

## 阶段 2：设计稿落地（Design）

**使用技能**：`.cursor/skills/pen-design-generator/`

1. 按照阶段 1 的 Spec 文档，调用 `pen-design-generator` 在 .pen 文件中实现设计。
2. 核心原则：
   - **组件优先**：ref 引用组件库组件 → 用 `descendants` **在插入时**填充业务文案（Spec 中「内容物料表」即文案来源；禁止保留组件占位文案如「一级菜单」「编号」）。
   - **对组件中的文案必须填充**：顶栏、侧栏、面包屑、页签、头信息、表格、卡片等凡含可读文案的组件，必须按 Spec 逐项替换。其中顶栏、**侧栏**、头信息、表格为多文案组件，插入时必须在 `I(..., { ref, descendants })` 中**一次性填齐**该组件内所有文本节点，不得遗漏。
   - **无匹配组件时**：才按规范从零创建，使用 Design Token
   - **不改组件尺寸**：除非 Spec 明确要求，否则保留组件库默认尺寸
3. 每完成一个主要区域后 `get_screenshot` 验证。

---

## 阶段 3：审核与修复（Review）

**参考文档**：`memory/pen-design-common-mistakes.md`

读取 `memory/` 目录下的常见错误文档，逐条对照当前设计稿检查：

| # | 检查项 | 快速判定 |
|---|--------|----------|
| 1 | 是否用了组件？ | 有顶栏/侧栏/表格/按钮等手搭的 → 替换为组件 ref |
| 2 | parent 是否字符串 ID？ | 检查 batch_design 调用记录 |
| 3 | text 节点是否写了 padding？ | 检查有无报错 |
| 4 | **组件文案是否已填充？** | 分析设计的组件，尤其是菜单栏、侧边栏、顶部导航栏等多层嵌套的组件，查看填充的文本是否符合设计意图 |
| 5 | 实例路径格式正确？ | `实例ID/devui:子节点ID` |
| 6 | imports 后是否 reload？ | 检查 get_editor_state 是否有 devui:xxx |
| 7 | 组件尺寸是否被不必要修改？ | 对比组件库默认值 |
| 8 | 单批操作是否 ≤25？ | 检查 batch_design 调用 |
| 9 | Copy 后是否用 descendants 改子节点？ | 避免事后 U() 已变 ID |
| 10 | 表头/列名是否业务语义？ | 截图确认无默认占位 |

发现问题 → 立即修复 → 修复后重新截图验证。

---

## 流程总结
你必须严格的执行完3个流程
```
用户需求
  │
  ▼
阶段 1：ui-page-spec-writer → 页面规范文档（Spec）
  │       ├ 区块拆解 + 内容物料表
  │       └ 组件选型表（映射到 devUI 组件 ID）
  ▼
阶段 2：pen-design-generator → .pen 设计稿
  │       ├ 组件优先（ref + 改文案）
  │       └ 每区域截图验证
  ▼
阶段 3：memory/ 常见错误审核 → 逐条检查 + 修复
  │
  ▼
交付设计稿
```
